# 会审后视频提示词编译模板

每个VID单独使用本模板。只读取已经通过 `templates/cinematic-department-review.md` 九项冲突闸门的结果；不得把部门讨论、冲突日志、完整状态账本或分镜长说明复制进生成端。

## 固定顺序

```text
【任务与输出】
[生成端/模式]；VID-xx；完整X秒；[单镜头/多镜头]；[画幅/分辨率等已确认参数]；[有/无对白、BGM、字幕、水印]。

【素材绑定】
图片1：[资产ID → CHAR/LOOK/PROP/SET/PAL/PANEL-ID-CLEAN → 锁定属性 → 适用USE镜头]
图片2：...
音频1：[VOICE/AUD-ID → 用途 → 适用时间]
[色卡图片必须写“REFERENCE-ONLY COLOR：只参考P编号、色彩分工、相对占比与调色方向，不渲染色块、文字、HEX、边框或版式”。]
[分镜参考只绑定用户或剧情选中的 `PANEL-ID-CLEAN`；不得绑定 `PAGE/PANEL-REVIEW`、无关格、SKIP格、箭头、节点、图例、文字或页面边框。]

【STYLE LOCK｜仅VID-xx】
[主模式和可观察的材质/曝光/颗粒/反差/空气/表演质感。把参考来源按色彩、光线、摄影、场景、表演、节奏或声音职能转译；不迁移参考作品剧情，不列无关参考名。]

【摄影合同】
[介质/传感器感；画幅；主镜头投影；焦段或FOV；光圈/景深；主体距离与焦点计划；快门/运动模糊意图；支撑/稳定方式；CAM起点→路径→速度→节点→停止构图。]

【场景、人物与空间】
[SET-ID与时间/天气；CHAR-ID/LOOK-ID和必要道具；世界坐标 → 摄影机坐标 → 屏幕坐标；前/中/后景；尺度主基准；遮挡链、允许露出部位与持续时间。]

【表演与生命感】
[PERF-ID；角色当下目标、阻碍、外在策略和潜台词；刺激→感知→决定→反应→新状态；当前景别可见的2–4项视线/呼吸/眨眼/吞咽/下颌肩颈/手指/重心信号；说话前气息、口型和说话后余韵；未说话者如何倾听。只写有因果的低幅度生命感，不写随机小动作。]

【时间轴】
0.0–t1秒：[第一视觉主体；触发→可见反应→唯一主动作/原台词；视线/呼吸/重心；摄影机/焦点；当段光线状态；主声音事件；段末状态。]
t1–t2秒：[同上。]
...
tN–X秒：[收束动作；台词已结束；人物/道具/摄影机/光影/声音最终状态。]

【环境光与人物光】
[LGT-ID；自然/实用/自发光/环境反射；世界位置、方向、软硬、相对强度；环境为什么可见；人物面部/眼睛或屏幕/头发/服装/轮廓如何可读；阴影、反射和雾尘如何响应。]

【色卡与调色】
[PAL-ID及色卡图片N；GUIDED或STRICT-N；P编号+HEX+绑定对象；主/辅/强调色占比；允许的受光变化；曝光/白平衡；GRADE-ID胶片或LUT作用；禁止色卡跨VID串入；禁止渲染色卡版式与标签。]

【妆造连续性】
[CHAR-ID/LOOK-ID；面部标记、发型发色、服装层、材质、配饰、义体、污损；动作过程中必须保持和允许变化的状态。]

【声音与混音】
[AUD/VOICE-ID；环境/动作/拟音/对白/非语言人声/BGM逐项允许状态；声源主体和空间位置；动作同步点；遮挡/远近/混响变化；对白、主动作、环境和音乐优先级；尾帧余韵或静默。]

【尾帧合同】
[最后0.5–1.5秒的构图、人物姿态/视线、道具、摄影机、焦点、光影、声音和下一VID继承方式。]

【高风险负面约束】
[只列当前VID最可能失败的3–8项：身份/妆造漂移、错误机位或投影、遮挡失败、色卡外有意色相、错误光源、台词/声音冲突、变形残留、尾帧错误等。]
```

## 编译规则

1. 每个VID只出现一次 `STYLE LOCK`、摄影合同、表演合同、色卡合同和声音合同；分段时间轴只写状态变化，不重复整段环境与风格。
2. 把摄影机、角色、焦点、光线和声音事件按时间排序；并发事件共享同一时间段，但只保留一个第一视觉主体。
3. 首次出现资产时写全名和绑定，后续使用稳定ID或无歧义简称；不得反复描述整套外观。
4. `STRICT-N` 只禁止新增有意设计色相，不禁止同一材质随光照产生合理明暗和饱和度变化。
5. “无人声”必须展开为对白、旁白和非语言人声的分别状态；“无BGM”不得误删环境和动作音。
6. 一镜到底写一条连续CAM路径；多镜头写清 `HARD CUT / MATCH CUT / OCCLUSION CUT` 等切点，不把多个镜头伪装为一条运镜。
7. 图片、视频和音频资产编号必须与当前VID执行素材清单及API `content` 顺序一致。
8. 删除重复画质词、无关创作者/作品名、上一VID残留、内部代码解释和不会改变画面的宣传语。
9. 已生成色卡时，同时引用图片N与关键P编号/HEX/对象；参考图预算不足时保留文字PAL合同，不挤掉必要角色板和执行分镜页。
10. 删除Markdown粗体、界面按钮/模型切换/积分/字符计数等UI残留；已确认的模型、时长、画幅和分辨率只在【任务与输出】出现一次。
11. 中英文同义词不连续堆叠；专业英文术语只在能减少歧义时保留一次。原台词、方言、音色和资产ID不因压缩而改变。
12. 将“8K”等超出实际输出的画质词转译为可观察的材质、纹理、边缘、高光和空间层次；不得把宣传分辨率当成平台能力。
13. 对幻想机械、变形或3D主体，不使用会删除主体的绝对NOT CGI/NOT 3D；改写为实体材质、实拍摄影观感或非塑料渲染等正向目标。
14. “人物生动”必须编译为目标、阻碍、刺激—反应、视线、气息、重心和手部等可见信号；每个短节拍只选择2–4项，不把眨眼、吞咽、摇头、抬眉、微笑和手势同时堆满。
15. 真人模式保留皮肤微纹理、受光层次、自然不对称和真实接触；3D模式保留身份化面部形变、眼神聚焦、呼吸、重量、碰撞及衣发次级运动。两种模式都禁止蜡像脸、表情瞬变、无因动作、脚底滑移和物体无接触移动。
16. REVIEW层的CAM/ACT/GAZE/FOCUS/LIGHT/READ路线先按节点与时间转译为自然语言摄影、调度、视线、焦点和光线变化，再从视觉输入中完全剔除；生成端只接收被选CLEAN单格与编译后的语义。
