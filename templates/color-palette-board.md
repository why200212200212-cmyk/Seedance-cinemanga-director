# 色卡生成、标注与视频引用模板

在场景、角色、妆造、世界光源和主要风格确定后生成色卡。色卡是当前场景或VID的 `REFERENCE-ONLY COLOR` 资产，不是剧情画面，不得替代角色板、场景板或执行分镜页。

## 1. 色卡源数据

```text
PAL-ID：[PAL-SEQ-xx / PAL-VID-xx]
作用域：[SEQUENCE / VID / SHOT]
模式：[GUIDED / STRICT-N]
场景/时间/天气：
调色目标：[主色、冷暖、饱和度、反差、黑位、高光、空气颜色]
GRADE-ID：[胶片/LUT/显示风格；只写当前作用域]

P01｜#RRGGBB｜颜色名｜职能｜绑定对象｜建议占比｜允许受光变化｜禁止迁移对象
P02｜#RRGGBB｜颜色名｜职能｜绑定对象｜建议占比｜允许受光变化｜禁止迁移对象
...
```

职能只能从以下选择：`固有色 / 自然光 / 实用光 / 环境光 / 阴影倾向 / 高光倾向 / 自发光 / 强调色 / 中性基底`。

先校验HEX为六位十六进制并去重。相同HEX承担多个对象时可以复用同一P编号，但要检查人物是否会与背景融合；需要轮廓分离时通过光照、明度或材质响应解决，不随意新增色相。

## 2. 标注色卡版式

优先使用宿主的排版、绘图或图像合成功能，以源数据确定性添加文字；不要依赖图像生成模型准确书写HEX和长标签。输出PNG时使用干净画布：

- 顶部：`PAL-ID｜场景/VID｜模式｜GRADE-ID`；
- 中部：按P01→PN排列等宽色块；颜色较多时分两行，不缩小到无法取样；
- 每个色块下：`Pxx`、`#RRGGBB`、短颜色名、短职能/绑定对象；
- 底部：主/辅/强调占比、环境光/人物光摘要、`REFERENCE-ONLY COLOR — DO NOT RENDER LABELS`；
- 背景使用中性浅灰或中性深灰，避免纯白界面压缩浅色差异；不得加入人物、场景缩略图、按钮、上传图标、加载浮层、光标或其他VID色卡；
- 导出前检查文字、编号、HEX、色块实际取样值和源数据一致；截图方式必须裁掉应用UI。

若宿主只能生成图像而不能可靠排字，先生成无字色块板，再使用可用的文本/排版工具叠加标签；若仍无法保证，保留结构化色卡表为唯一真源，不伪造“已准确标注”的图片。

## 3. 图片提示词（仅生成无字底板时）

```text
生成一张专业电影制作色彩脚本板，画幅[16:9/21:9]，干净中性[浅灰/深灰]背景，共[N]个等宽纯色色块，按[P01→PN]从左到右、从上到下排列。色块边界清晰、无渐变、无纹理、无人物、无场景、无装饰图标、无UI、无加载浮层、无水印。保留规则的标题区、标签区和底部说明区为空白，文字与HEX将在后期确定性叠加。
```

图像模型生成的色块颜色仍需按源数据重新填充或采样校正，不能把视觉近似当作HEX验证。

## 4. 执行素材绑定

当前VID素材清单中登记：

```text
图片N｜PALETTE-PAL-VID-xx｜REFERENCE-ONLY COLOR｜适用VID/Sxx｜PNG URL/asset URI
用途：只参考P01–PN的色彩分工、相对占比、冷暖和调色方向；不参考色卡排版、文字、边框或背景。
```

- 每个VID只绑定当前作用域的必要色卡；禁止上传其他段落色卡或包含多场景色板的截图；
- 图片编号与API `content` 顺序一致；色卡被删除、替换或重排后重新生成编号清单；
- 参考图数量受限时，优先保留必要角色板、当前执行分镜页、关键场景/首尾帧；色卡可降级为文字PAL合同，不挤掉身份与空间锚点；
- 色卡不是首帧或内容图，不要求成片出现色条、HEX、标题或白色画布。

## 5. 视频提示词引用格式

在 `templates/compiled-video-prompt.md` 的素材绑定和色卡/调色两处分别写：

```text
图片N（PAL-VID-xx，REFERENCE-ONLY COLOR）：仅参考色彩分工、相对占比与调色方向，不渲染色块、文字、HEX、边框或版式。

【色卡与调色】
应用图片N的PAL-VID-xx，模式STRICT-13：P01 #F0B670只用于金色直射光；P02 #C8581F只用于落日天空；……。允许同一材质随曝光、阴影、雾和胶片响应发生合理明度/饱和度变化；禁止引入未登记的有意设计色相。GRADE-ID只作用于当前VID。
```

GUIDED模式把“禁止引入”改为“其他颜色保持在既定色相家族内，不形成新的视觉主色”。不要只写“严格按图片N”，必须同时写出关键P编号、HEX、绑定对象和允许变化。

## 6. 色卡质检

- [ ] PAL-ID、作用域、模式和GRADE-ID明确；
- [ ] 每个P编号、HEX、颜色名、职能、绑定对象和占比可追溯；
- [ ] 标注图色块取样与源数据一致，文字无错码；
- [ ] 图片无UI、加载浮层、人物、场景和其他VID颜色；
- [ ] 环境光色、人物固有色和自发光没有混成同一职能；
- [ ] 正文没有未解释的色卡外HEX；
- [ ] 视频提示词同时引用图片N和关键色号，且明确不渲染色卡版式；
- [ ] API素材顺序与提示词编号一致；参考图预算不足时已按身份/空间优先级安全降级。
