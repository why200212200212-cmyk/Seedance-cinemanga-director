# Seedance 2.0 详细分镜图模板

剧情分析完成后使用本模板。先选择分镜形式并锁定必要角色，再生成分镜图；不得为了填满固定宫格增加、删除或拆碎剧情。

## 分镜规划单

```text
【剧情节拍】
[节拍01 → 节拍02 → ……]

【当前视频条目与时间轴】
- VID-ID：[VID-01]
- 本条总时长T_video：[4.0–15.0秒；最长15.0秒]
- 执行时间轴：[0.0–T_video]
- 分板边界：[本板只服务当前VID；超过15秒的内容拆为下一VID并另建执行板]

【分镜形式】
[标准故事板 / 九宫格 / 二十五宫格 / 智能宫格]

【选择依据】
- 当前条总时长：[X.X秒]
- 必要镜头数：[N]
- 连续镜头关键状态数：[N]
- 动作复杂度：[低 / 中 / 高]
- 对白与时间预算：[说明]
- 连续性风险：[角色 / 道具 / 空间 / 光影 / 尾帧]
- 排版方案：[行列、分页、重点格放大方式]
- 灵活选择说明：[板式由时长+节拍+复杂度共同决定；不按秒数机械绑定]

【阅读顺序】
S01 → S02 → …… → SNN

【镜头使用清单】
- PANEL-VID01-01｜S01｜USE｜[第几条、时间范围、必须生成原因、绑定参考图]
- PANEL-VID01-02｜S02｜REFERENCE-ONLY｜[服务哪个USE镜头、参考内容、不占生成时长]
- P03｜SKIP｜[弃用原因；不进入执行分镜图、不提交生成端]

【双层分镜资产】
- CLEAN-PAGE-ID：[PAGE-VID01-CLEAN；无箭头、无文字、无边框内制作标注]
- REVIEW-PAGE-ID：[PAGE-VID01-REVIEW；同构复制CLEAN画面并叠加CAM/ACT/GAZE/FOCUS/LIGHT/READ]
- 单格资产规则：[每个USE格导出PANEL-ID-CLEAN；REVIEW单格只供审阅，禁止提交视频生成端]

【单格选择映射】
- PANEL-VID01-01｜S01 / S01-K01｜CLEAN-ASSET-ID｜服务VID与时间段｜主首帧/构图/动作/尾帧用途
- PANEL-VID01-02｜S02 / S02-K01｜CLEAN-ASSET-ID｜服务VID与时间段｜用途

【角色板绑定】
- CHAR-001｜角色名：[独立角色板/图片1；固定面部、发型、体型、服装；适用镜头]
- CHAR-002｜角色名：[独立角色板/图片2；固定面部、发型、体型、服装；适用镜头]

【场景板绑定】
- 场景01：[空间结构、轴线、主光、时间、天气]

【部门合同绑定】
- STYLE-ID / PAL-ID / GRADE-ID：[当前VID作用域与禁止继承项]
- LOOK-ID：[逐角色妆面、发型、服装层、材质、配饰/义体与污损]
- PERF-ID：[逐角色目标、阻碍、策略、潜台词、触发与可见反应]
- LGT-ID：[世界光源位置、方向与环境光/人物光职责]
- AUD-ID / VOICE-ID：[声源主体、空间、台词/非语言人声/BGM状态与混音优先级]
```

- PANEL-ID / CLEAN-ASSET-ID / REVIEW-ASSET-ID：

- 时间闭合：[首镜从0.0秒开始；本镜结束时间=下一USE镜头开始时间；末镜结束=T_video且≤15.0秒]
- 画面：
- 景别/机位/角度/焦段感/画幅：
- 构图与前中后景：
- 世界坐标 → 摄影机坐标 → 屏幕坐标：
- 尺度主基准 / 遮挡链：
- 角色ID/角色名/独立参考图编号/站位/朝向/视线/表情：

- 摄影机运动与焦点：
- 路线代码：[CAM-A / ACT-A / GAZE-A / FOCUS-A / LIGHT-A]

## 分镜图生成提示词

```text
生成一套高分辨率、同构双层影视制作分镜资产，只服务当前视频条目[VID-01]。先生成 `PAGE-VID01-CLEAN`：每个PANEL画面清楚、无文字、无箭头、无节点、无边框内制作符号；再复制相同画格与裁切，使用确定性排版工具生成 `PAGE-VID01-REVIEW`，叠加CAM/ACT/GAZE/FOCUS/LIGHT/READ与信息栏。不得依赖图像模型准确书写路线代码和长文字。每个USE格同时导出独立 `PANEL-ID-CLEAN`，供后续单格选择与视频参考；审阅标注板和REVIEW单格永不提交视频生成端。

本条总时长[T_video=X.X秒]，范围必须为4.0–15.0秒，最长15.0秒；执行时间轴为0.0–X.X秒。采用[标准故事板/九宫格/二十五宫格/智能宫格]，共[N]个必要镜头或关键状态。板式由本条时长、剧情节拍、必要镜头/关键状态数和动作复杂度灵活选择，不按秒数机械绑定：短视频的复杂动作可使用九宫格、二十五宫格或智能宫格，15秒简单镜头可使用标准故事板。[横向/纵向]阅读，镜头编号从S01连续到SNN；REVIEW页格间使用明确阅读箭头，CLEAN页保持纯画面。多页时页码和镜头号连续，但仍共享当前VID的同一条0.0–T_video时间轴。超过15秒的内容不得放在本板，必须拆为下一VID并另建执行板。

REVIEW页头清晰显示 `VID-01｜T_video=X.Xs｜0.0–X.Xs｜MAX 15.0s｜当前板式`。REVIEW页每格左上角显示镜头序号与USE/REFERENCE-ONLY状态；右上角同时显示 `VID-ID`、逐镜起止时间 `t_start–t_end` 和单镜时长 `Δt`。所有USE镜头按阅读顺序从0.0秒覆盖到T_video，末镜结束不超过15.0秒；不得无解释重叠、倒序或把REFERENCE-ONLY计入独立时长。同一连续镜头的关键状态格使用Sxx-Kxx和 `at t=X.Xs`，不伪装成新增剪切。REVIEW页格下信息栏写景别、机位、焦段感、角色ID、动作、运镜、台词/声音、道具、光影、转场和连续性锚点。SKIP镜头不得出现在本执行分镜图中，只放入单独弃用清单。REVIEW页固定提供AI可辨识路线图例：CAM摄影机运动=金色粗实线双箭头，ACT角色/物体运动=青色实线单箭头，GAZE视线=白色细虚线空心箭头，FOCUS焦点变化=蓝色点划线菱形端点，LIGHT主光=橙色放射宽箭头，READ阅读顺序=页边绿色编号箭头。不得只依赖颜色，每条线重复写代码；CLEAN页及CLEAN单格不得出现这些制作标记。

所有路线只出现在REVIEW层。CAM路线使用连续编号节点，例如CAM-A0、CAM-A1、CAM-A2、CAM-A-END；转向、升降、变速和停顿处必须设节点。每个节点间重复方向箭头并标时间、动作类型、速度和机位高度；转弯处同时画进入箭头与沿新方向离开的箭头，明确到达节点后的继续方向。不能只在整条线末端放一个箭头。复杂三维路线增加俯视或侧视小图，与主画面共享相同节点ID。路线暂停使用HOLD节点，恢复时从HOLD节点绘制新箭头。视频编译时读取这些节点并转成自然语言，随后从视觉参考中剔除路线层。

复杂或交叉路线使用CAM-A、CAM-B分组，必要时增加局部放大格。所有标注位于边框或信息栏，不遮挡人脸、手部、关键道具和动作路径。

角色严格继承已绑定的独立多视角角色板和LOOK-ID；每个人像附近显示永久角色ID，格下信息栏重复角色名和独立参考图编号。同一角色跨格不变脸，不同角色不混脸。场景结构、轴线、主光方向、PAL/LGT作用域、道具数量和持有关系保持连续。每格把世界坐标、摄影机坐标、屏幕坐标、尺度主基准和遮挡链写在信息栏，不让标注遮挡画面。只呈现剧情分析确认的镜头，不添加装饰性人物、道具或剧情。

[逐格粘贴“每格镜头卡”的画面与执行信息。]
```

## 输出前检查

- 镜头号、时间码、条目映射和阅读顺序无歧义；
- 当前执行板只对应一个VID，页头已标4–15秒总时长、0.0–T_video时间轴和MAX 15.0s；
- 每个USE格已标起止时间和Δt；首镜从0.0秒开始，末镜在T_video结束，时间顺序闭合且不超过15.0秒；
- 连续多条逐条另建执行板并从0.0秒重新计时；跨条总览已标REFERENCE-ONLY OVERVIEW，未代替执行板；
- USE、REFERENCE-ONLY和SKIP清单明确；执行图不含SKIP，REFERENCE-ONLY写明服务对象且不占生成时长；
- CLEAN与REVIEW双层资产同构；每个USE格都有稳定PANEL-ID和独立CLEAN单格；
- 四种分镜形式均保留完整字段，未因格数增加而缩减信息；
- CAM、ACT、GAZE、FOCUS、LIGHT、READ图例完整，任何路线在不看颜色时仍可识别；
- 摄影机路线具有明确起点、终点、归属、方向、速度阶段、机位高度和停止构图；
- 每次转向、升降、变速或停顿均有连续节点，每一段都有箭头和下一方向，俯视/侧视小图与主画面节点一致；
- 所有原台词逐字一致；
- 分镜格数等于必要画面节点数，不为填格改变剧情；
- 每格均可映射到 Seedance 2.0 提示词中的一个镜头或关键状态；
- 角色板、场景板、上一条尾帧和当前分镜页的图片编号绑定清楚；
- 选择某一格生成视频时，已映射唯一PANEL-ID、Sxx/Sxx-Kxx、VID时间段和CLEAN-ASSET-ID；只提交干净单格，REVIEW板、箭头、文字、节点和图例全部排除；
- 每个角色拥有单独角色板和永久ID，多人总览图没有被当作身份参考图；
- 分镜标注只服务制作参考，不成为最终视频字幕、箭头、轨迹线、节点、边框或画面文字。
