# 最终质量检查清单

输出前逐项检查。任一项不通过，先修正再提交。

## A. 剧情与人物

- [ ] 已先完成剧情分析，再按剧情需要选择标准故事板、九宫格、二十五宫格或智能宫格；
- [ ] 分镜形式与实际镜头数量和信息密度匹配，没有为凑固定格数新增或拆碎剧情；
- [ ] 标准故事板、九宫格、二十五宫格或智能宫格按当前条4–15秒总时长、节拍、必要镜头/关键状态数和复杂度灵活选择，没有按秒数机械绑定；
- [ ] 每张执行分镜板只对应一个VID；连续多条逐条另建执行板，跨条总览只标REFERENCE-ONLY OVERVIEW；
- [ ] 标准故事板、九宫格、二十五宫格和智能宫格均使用相同完整字段；信息过密时已扩大画布或拆页；
- [ ] 每镜标记USE、REFERENCE-ONLY或SKIP，并映射到具体视频条目和时间段；
- [ ] 执行分镜图、最终提示词和提交生成端的参考图不含SKIP镜头；REFERENCE-ONLY明确服务对象且不占独立生成时长；
- [ ] 所有原剧情必需动作和台词均至少映射到一个USE镜头；
- [ ] 分镜图镜头序号连续、阅读顺序明确，每格包含 Seedance 2.0 执行所需的镜头、角色、动作、声音、光影和连续性信息；
- [ ] 分镜标注位于边框或信息栏，没有遮挡人脸、手部、道具和动作路径；
- [ ] 每页包含CAM、ACT、GAZE、FOCUS、LIGHT、READ路线图例，颜色、线型、箭头头型和代码均可区分；
- [ ] 每条运镜路线都有起点、终点、方向、机位高度、速度阶段、轴线侧和停止构图；
- [ ] 转向、升降、变速和停顿点使用连续编号节点，每个节点间都有重复箭头、时间段和动作说明；
- [ ] 转弯处同时显示进入方向与离开方向，没有只在整条路线末端放一个箭头；
- [ ] 复杂三维路线的俯视/侧视小图与主画面共享相同节点ID和方向；
- [ ] 交叉路线使用独立ID或局部放大格，没有无归属、穿过主体或只靠颜色判断的箭头；
- [ ] 每个必要角色都有独立的正面、左侧面、右侧面和背面设计；
- [ ] 每个角色单独拥有一张运行时角色板、永久角色ID和独立参考图编号，没有用多人合并图代替；
- [ ] 角色身份注册表完整，分镜、图片提示词、视频提示词与API素材顺序使用同一绑定；
- [ ] 每个分镜格在人像附近标角色ID，信息栏重复角色名与参考图编号，多人同框时逐人绑定；
- [ ] 同一角色跨视角保持同一身份；
- [ ] 不同角色已完成两两面部差异检查，不依赖发型或服装作为唯一识别手段；
- [ ] 除剧本明确要求双胞胎、克隆体等例外外，没有相同或高度相似的模板脸；
- [ ] 剧情相似例外已列出可辨识差异，且没有自行虚构亲缘或相似关系；
- [ ] 剧情因果和事件顺序与原文一致；
- [ ] 原台词逐字一致；
- [ ] 没有新增无关人物、动物、场景、道具和宏观动作；
- [ ] 当前段落必须出现的人物全部存在；
- [ ] 无功能角色已排除；
- [ ] 台词说话人和声音来源正确。

## B. 时间与节奏

- [ ] 单条模式总时长严格15秒；
- [ ] 多条模式每条4–15秒；
- [ ] 每张执行板页头显示VID、T_video、0.0–T_video时间轴和MAX 15.0s，板内总时长没有超过15秒；
- [ ] 每个USE镜头显示起止时间和单镜时长Δt；首镜从0.0秒开始，末镜在T_video结束；
- [ ] USE时间按阅读顺序闭合，无解释重叠、倒序或超时；REFERENCE-ONLY和SKIP不占独立生成时长；
- [ ] 同一连续镜头的关键状态格使用Sxx-Kxx与时点标记，没有误写成额外剪切；
- [ ] 台词朗读时长合理；
- [ ] 停顿、呼吸、反应和动作有时间；
- [ ] 最后0.5–1.5秒用于收束；
- [ ] 结尾没有卡住半句台词。

## C. 镜头与表演

- [ ] 景别和机位服务剧情；
- [ ] 运镜路径物理可行；
- [ ] 人物站位、面对关系和屏幕方向明确；
- [ ] 动作顺序具体可执行；
- [ ] 手部状态和道具使用关系正确；
- [ ] 无穿墙、穿模、瞬移和无依据慢动作；
- [ ] 焦点变化与主体动作匹配。

## D. 连续性

- [ ] 角色脸型、发型、服装、体型一致；
- [ ] 场景结构、门窗和家具位置一致；
- [ ] 道具数量、位置、朝向和状态一致；
- [ ] 左右手没有无故交换；
- [ ] 光源方向、色温、阴影和反光一致；
- [ ] 人物视线和屏幕方向一致；
- [ ] 未发生无解释越轴；
- [ ] 多条模式的起始帧完整继承上一条尾帧。
- [ ] 每个USE镜头中的必要角色ID都能唯一反查到独立角色板；
- [ ] 每张执行分镜页只绑定一个VID，并明确覆盖的USE镜头；
- [ ] 执行素材绑定清单、视频提示词【图片绑定】和API content使用完全相同的图片顺序；

## E. 声音与纯净度

- [ ] 环境音和动作音与画面同步；
- [ ] 默认无音乐、无BGM；
- [ ] 默认无字幕、无画面文字；
- [ ] 默认无水印、无Logo；
- [ ] 内心独白人物不张嘴；
- [ ] 电话、门外、广播和远处声源有空间属性。

## F. 输出格式

- [ ] 使用正确模板；
- [ ] 单条模式包含 `_::~OUTPUT_START::~_` 和 `_::~OUTPUT_END::~_`；
- [ ] 多条模式每条包含起始继承和尾帧接力合约；
- [ ] 图片提示词是静态代表画面；
- [ ] 视频提示词是连续动态描述；
- [ ] 没有输出隐藏推理或无关解释。

## G. 影视真实感硬门槛

- [ ] 每个动作都有触发、路径、受力和落点，没有无因提前反应；
- [ ] 每个运镜都有唯一主要目的、可实现路径和明确停止构图；
- [ ] 焦段、主体距离、景深、拉焦与画面透视不冲突；
- [ ] 主光、高光、阴影、反射、雨雾和体积光都有可解释来源；
- [ ] 人物重心、脚底接触、手与道具接触、衣发次级运动符合物理；
- [ ] 表情由视线、呼吸、面部和姿态微动作组成，不是僵硬摆拍；
- [ ] 没有用画质词堆叠替代具体摄影、表演和美术指令；
- [ ] 单镜复杂度在可控范围，必要时已经拆镜或拆条。

## H. 模式专项

### 3D国漫影视级

- [ ] 角色身份锚点、服装层、材质参数和纹样位置稳定；
- [ ] 皮肤、头发、布料、金属和玻璃的材质响应互不混淆；
- [ ] 动作含预备、发力、位移、缓冲和余韵，次级运动合理衰减；
- [ ] 无脸型漂移、材质闪烁、几何呼吸、肢体穿模和道具悬浮。

### 真人影视级

- [ ] 摄影设备、焦段、机位、路径和现场空间相容；
- [ ] 曝光、运动模糊、景深和焦点变化符合镜头条件；
- [ ] 肤质、妆发、服装、汗水、雨水、污损和伤势状态连续；
- [ ] 无蜡像脸、脚底滑行、果冻形变、焦点抽搐和曝光泵动。

## I. 定点修订与重生成

- [ ] 每个可生成图片、视频、分镜页和尾帧都有稳定资产ID、当前版本和可反查的显示别名；
- [ ] 用户反馈已定位到唯一图片、VID、镜头号或时间段；无法唯一定位时只提出一个最小澄清问题；
- [ ] 已保留用户原话、问题诊断、必须修改项和必须锁定项，没有扩大修订范围；
- [ ] 生成前已展示新版完整提示词、素材绑定、负面约束和影响范围，状态为 `AWAITING_USER`；
- [ ] 用户能够直接修改提示词；没有明确“继续生成”或等价授权时未调用生成工具；
- [ ] 图片只重做指定资产；平台不支持视频局部编辑时只重做对应VID，没有重做整个项目；
- [ ] 旧版本保留到新候选通过质检，新版本失败时没有覆盖已接受版本；
- [ ] 角色板、场景板、分镜页、尾帧或结束状态改变产生的下游影响已标 `REVIEW_REQUIRED`，没有未经批准连锁生成；
- [ ] 新候选先验证用户指出的问题，再检查锁定项、模式专项和前后连续性。
