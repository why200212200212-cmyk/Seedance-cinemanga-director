## 一、竖屏与横屏的本质差异

| 维度 | 横屏（16:9） | 竖屏（9:16） | 差异影响 |
|---|---|---|---|
| 横向空间 | 宽（可容纳多人） | 窄（只容1-2人） | 竖屏不适合大场面 |
| 垂直空间 | 有限 | 极丰富 | 竖屏利用垂直调度 |
| 视线重心 | 水平左右 | 垂直上下 | 运镜以纵摇/升降为主 |
| 信息密度 | 可大景多人 | 只能聚焦单一主体 | 更依赖特写和中景 |
| 沉浸感 | 旁观感 | 亲密/压迫感 | 距离更近、情绪更烈 |

### 核心认知转变
**竖屏不是"把横屏上下裁掉"——而是"用垂直重新构思每一镜"**

横屏的经典构图（如地平线在1/3处）在竖屏中不成立。竖屏的构图必须**利用高度差做权力隐喻**。

## 二、竖屏权力构图（Power Composition）

### 2.1 垂直权力轴

**高位（画面上方1/3区）**：
- 权力者、压迫者、全知者
- 低角度仰拍 → 具视觉统治力

**中位（画面中部1/3区）**：
- 对话者、平等关系、观察者

**低位（画面下方1/3区）**：
- 受害者、屈从者、受困者
- 高角度俯拍 → 具视觉脆弱感

### 2.2 文档中的权力构图
```
高位（画右远处）：索菲亚——大祭司袍 + 权杖顿地
中位（画右中景）：莉拉——红裙站立，俯视下方
低位（画左地表）：奈雅——瘫软、跪伏、脸贴冰霜
```

**三角形欺压几何**：母亲(高位)+长姐(中位)+女主(低位)构成稳固的压迫三角形。

### 2.3 AI提示词
```
vertical composition, power hierarchy, high angle on victim at bottom, low angle on oppressor at top
```

## 三、竖屏的六种构图策略

### 3.1 对称式垂直构图
- 人物居中，利用对称构图产生庄重压迫感
- **适用**：权力展示、宗教仪式、审判场景
- **AI**：`symmetrical composition, centered subject, vertical frame`

### 3.2 低角度仰拍构图
- 摄影机贴地仰拍，人物占满上方2/3
- **适用**：权力者登场、巨人视角、英雄时刻
- **AI**：`low angle, looking up, subject dominating top half of frame`

### 3.3 高角度俯拍构图
- 从上往下拍，人物被压缩在画面底部
- **适用**：受辱、绝望、躺地、跪伏
- **AI**：`high angle, looking down, subject trapped in bottom third`

### 3.4 引导线垂直构图
- 利用柱状、垂直线条将视线引向上下
- **适用**：牢笼、走廊、瀑布、高楼
- **AI**：`vertical leading lines, framed by columns/iron bars, vertical depth`

### 3.5 框架式竖幅构图
- 利用门框、窗口、链子形成竖画幅内的次级框
- **适用**：囚禁视角、偷窥视角、限制空间
- **AI**：`framed by dungeon window, character inside vertical frame, claustrophobic`

### 3.6 极浅景深构图
- 竖屏横向空间窄，浅景深可以进一步聚焦
- **适用**：情绪大特写、微表情捕捉
- **AI**：`extremely shallow depth of field, only face in focus, bokeh everything else`

## 四、竖屏运镜的垂直化改造

### 4.1 核心原则
**横屏以水平运镜为主 → 竖屏以垂直运镜为主**

| 横屏运镜 | 竖屏改造方案 | 案例 |
|---|---|---|
| Pan（横摇） | 重新设计为 Tilt（纵摇） | 展现人物全貌时用纵摇取代横摇 |
| Tracking（横移跟随） | 升级为 Crane/Boom（升降跟随） | 角色上楼梯、跪→站等垂直运动 |
| Wide Shot | 替换为分层Z轴深景 | 前景虚+中景主体+后景环境 |
| Dolly In | 最佳竖屏运镜（Z轴推入） | 微距推入深入情绪 |
| 双人面对面 | 转换为上下垂直+内反拍 | 高度权力差取代左右平等 |

### 4.2 文档中的竖屏运镜设计
- "利用竖屏9:16的纵深，极度克制地压缩人物横向生存空间，制造窒息感"
- "默认使用中长焦，将人物从环境里剥离，强化微表情"
- "滑轨低位快切跟随女主的踉跄脚步"

## 五、竖屏的景深层次（Z轴调度）

竖屏的空间调度主要依靠Z轴（景深）而非X轴（横向）：

| 层次 | 画面区域 | 内容 | 分辨率 | 功能 |
|---|---|---|---|---|
| 前景 | 底部/模糊 | 焦石/铁链碎屑/水晶 | 低（虚化） | 空间边界、遮挡过渡 |
| 中景 | 中部/清晰 | 人物主体/核心动作 | 高（焦点） | 叙事重心 |
| 后景 | 顶部/远景 | 环境/权力者/天空 | 中 | 空间锚定、权力坐标系 |

**文档中的Z轴设计（SB02a）**：
```
Z轴：前景虚化铁窗轮廓 → 中景祭坛割脉 → 后景冰霜石墙
```

## 六、竖屏对AI生成的特殊要求

### 6.1 画幅确认
所有Prompt必须以 `9:16 vertical, portrait aspect ratio` 或 `vertical format` 开头提示AI

### 6.2 中长焦默认
竖屏中默认使用中长焦（85mm-135mm等效），避免广角导致的信息过载：
```
85mm lens, medium telephoto, character isolated from background
```

### 6.3 人物构图锚定
竖屏中人物容易"飘移"，必须在Prompt中写死人物在画面中的位置：
```
Naya positioned in frame lower half, face in center, Lila in frame upper right corner
```

### 6.4 群像构图限制
竖屏不适合3人以上同框。如需多人出现在同一镜头：
- 利用垂直叠加（一人跪在低位+一人站在中位+一人站在高位）
- 或利用景深叠加（前/中/后景各一人）

## 七、竖屏短剧的经典场景构图

### 7.1 地牢/囚禁场景
```
构图：垂直分割
上1/3：铁窗透光（希望的幻觉）
中1/3：祭坛+冰链（受虐空间）
下1/3：血迹+冰霜（痛苦证据）
```

### 7.2 宫殿/权力场景
```
构图：低角度仰拍
上2/3：王座+施暴者（绝对权力）
下1/3：跪着的受害者（绝对服从）
```

### 7.3 逃亡/追逐场景
```
构图：后拉镜头
上1/3：追逐者逼近（威胁持续）
中1/3：奔跑主体（核心焦点）
下1/3：地面障碍（环境压迫）
```

### 7.4 亲密/微表情场景
```
构图：极浅景深 ECU
全屏：只有一只眼/一个嘴唇/一滴泪
背景：完全虚化
情绪：极致的生理战栗
```

## 八、竖屏的三段式情绪节奏

### 8.1 开场（0-10s）
- 宽+高 -> 建立空间 > 微距 -> 切入角色
- **视觉策略**：速建空间锚，然后快速切入角色的生理状态

### 8.2 中段（10-100s）
- 交替对话 + 微距生理冲击
- **视觉策略**：三角形权力外反拍 + 内反拍微表情捕捉

### 8.3 高潮（100-130s）
- 极致压迫 + 运动快切
- **视觉策略**：高密度快切 + 垂直权力强化 + **微距硬锁**

## 九、竖屏构图检查清单

```
□ 1. 本镜中人物的垂直权力位置是否明确？
□ 2. 横向空间是否被压缩至窒息？
□ 3. Z轴（景深）是否被充分利用？
□ 4. 是否默认使用中长焦剥离人物与环境？
□ 5. 三层次（前虚/中主/后环境）是否齐备？
□ 6. 如果是多人同框，是否垂直叠加或景深叠加？
□ 7. 如果是对话，是否遵守内反拍/外反拍/三角形外反拍？
□ 8. 9:16画幅是否在Prompt中已声明？
```
