# 定点修订与重生成协议

只在用户对已生成图片、视频、分镜格或局部时间段提出不满意反馈时读取。此协议接在成片/图片复核之后，不改变前面的剧情分析、角色注册、自适应分镜、双影视引擎和连续性流程。

## 1. 稳定定位

每个可生成资产必须拥有稳定逻辑ID；版本变化时ID不变，只递增修订号：

| 资产 | 稳定ID示例 | 版本示例 |
|---|---|---|
| 独立角色板 | `IMG-CHAR-001` | `R01` |
| 场景板 | `IMG-SCENE-001` | `R01` |
| 执行分镜页 | `IMG-VID-02-PAGE-01` | `R01` |
| 单独分镜格/关键帧 | `IMG-VID-02-S03` | `R01` |
| 视频条目 | `VIDEO-VID-02` | `R01` |
| 接力尾帧 | `IMG-VID-02-TAIL` | `R01` |

资产索引同时登记显示顺序、人物/场景名、VID、PAGE-ID、镜头号、时间段和当前版本，使“图3”“角色A那张图”“第二条视频6秒处”等表达能够反查唯一ID。

按以下优先级解析用户目标：

1. 稳定资产ID或VID/PAGE-ID；
2. 镜头号与时间段；
3. 当前回答中的图片/视频显示序号；
4. 角色名、场景名、剧情动作或相邻资产描述；
5. “上一张”“刚才那条”等会话相对位置。

若用户直接上传一张尚未登记的图片或视频，将其作为已接受的导入基线登记为 `IMG-IMPORT-001` 或 `VIDEO-IMPORT-001｜R01`，记录附件/文件定位与用户描述，再进入同一修订流程；不得因缺少旧项目索引而重做其他资产。

若仍对应多个目标，只问一个问题并列出候选ID；不得猜测后直接生成。

## 2. 修订单

为每次反馈创建 `REV-001` 起的修订单，记录：

- `TARGET`：稳定资产ID、当前版本、VID/PAGE/镜头/时间段；
- `USER_FEEDBACK`：保留用户原话，不擅自扩大问题；
- `DIAGNOSIS`：身份、构图、动作、运镜、表演、物理、光影、声音、文字污染、连续性或其他；
- `CHANGE`：这次必须改变的可见结果；
- `LOCK`：剧情、原台词、人物身份、已接受外观、服装、空间、道具、前后状态及其他不允许变化项；
- `IMPACT`：仅目标 / 当前VID / 后续连续性待复核；
- `STATUS`：`DRAFT → AWAITING_USER → APPROVED → GENERATING → CANDIDATE → ACCEPTED/REJECTED`。

不要把“更好看”“更电影感”直接当作提示词。将反馈转译成能够改变结果的摄影、表演、美术、物理或时间线指令；不确定审美方向时给出最多两个清晰方案供用户选择。

## 3. 先提示词，后生成

始终先按 `templates/revision-preview.md` 输出：

1. 已定位目标与当前版本；
2. 问题诊断；
3. 修改项和锁定项；
4. 新版完整图片或视频提示词，不只给差异片段；
5. 图片绑定与必要负面约束；
6. 影响范围和预计生成单位；
7. 明确提示用户可直接编辑，或回复“继续生成”。

输出预览后停止，状态保持 `AWAITING_USER`。用户只修改提示词但未要求生成时，合并其修改并再次展示完整版本；用户明确要求“修改后直接生成”时，可将修改视为同一条消息中的批准。

## 4. 最小重生成单位

### 图片

- 角色板不满意：只重做该角色板，锁定永久 `CHAR-ID` 和用户未要求改变的身份锚点；
- 场景板不满意：只重做该场景板，锁定空间拓扑和已接受锚点；
- 执行分镜页不满意：只重做该页；若只是一格，优先生成该格的独立修订图，再按用户要求决定是否回填整页；
- 已有原图且宿主支持图像编辑时，把原图作为编辑参考并明确保留区；不支持编辑时生成新候选，不声称像素级局部修改。

### 视频

- 整条不满意：只重做对应 `VIDEO-VID-xx`；
- 某镜头或某秒不满意：锁定问题时间段、镜头号、前一状态和后一状态，只改该段提示词；若当前生成端不能局部编辑，仍只新建该VID的完整候选版本，不重做其他VID；
- 不把重生成当作对原任务的覆盖写入。每次创建都是新候选任务；旧任务ID与旧成片保留到新版本通过质检。

## 5. 依赖影响

只标记受影响资产，不自动连锁生成：

- 角色板改变：引用该 `CHAR-ID` 的分镜页和视频标 `REVIEW_REQUIRED`；
- 场景板改变：引用该场景的VID标 `REVIEW_REQUIRED`；
- 分镜页改变：对应视频提示词、素材顺序和 dry-run 标 `REVIEW_REQUIRED`；
- 视频中间段改变但结束状态不变：后续VID保持有效；
- 视频尾帧、角色最终状态、道具、位置、光影或声音桥改变：下一VID起始继承标 `REVIEW_REQUIRED`；
- 仅修画质噪点且身份、构图和状态均不变：下游可保持有效，但仍记录理由。

用户批准下游更新前，只报告影响和建议顺序，不产生新的付费任务。

## 6. 候选验收与版本提升

新候选生成后：

1. 先检查用户指出的问题是否真的改善；
2. 再检查 `LOCK` 项、当前模式专项门槛、素材绑定和前后连续性；
3. 通过后把候选版本标为 `ACCEPTED`，旧版标为 `SUPERSEDED` 但保留记录；
4. 未通过时标为 `REJECTED`，继续使用旧版，不无限自动重试；
5. 需要再次修改时新建下一修订号，并重新走提示词预览与用户批准。
