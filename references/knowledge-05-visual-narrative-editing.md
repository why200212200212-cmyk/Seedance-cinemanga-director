## 一、180度轴线规则（The 180-Degree Rule）

### 1.1 定义
在两个角色对话或对峙时，两个角色之间形成一条虚拟的"轴"。摄影机必须保持在轴的同一侧（180°弧内）。

### 1.2 为什么要遵守
- 观众对空间方向的认知依赖连续性
- 越轴会使A和B在画面中的左右关系突然反转
- 观众在0.2秒内就会意识到"不对"

### 1.3 越轴的后果（AI特别容易踩的坑）
```
正反打A（A在左/B在右） → 越轴 → 正反打B（A在右/B在左）
= 观众感觉："他们俩换位置了？"
```

**AI是如何犯错的**：AI生成第1镜（A左B右），生成第2镜（B左A右），因为在模型中"左右"没有连续性概念。

### 1.4 合法越过轴线的方法

五种方式（按最优到最可行排序）：

| 方法 | 说明 | AI实现难度 |
|---|---|---|
| 1. 移动镜头 | 摄影机在单次运镜中穿过轴线 | ★★★★（最难） |
| 2. 中性镜 | 插入一个正面轴向镜头重置空间感 | ★★（容易） |
| 3. 重新建立全景 | 用一个全景重新确立空间关系 | ★★★ |
| 4. 角色移动 | 让角色走动改变轴线方向 | ★★★★ |
| 5. POV重置 | 切换为主观视角 | ★★★★ |

**AI推荐方案**：在分镜设计中始终保持轴线不越，用中性镜或全景重建立代替越轴。

## 二、30度规则（The 30-Degree Rule）

### 2.1 定义
两个同景别镜头之间，摄影机角度至少改变30度。

### 2.2 为什么重要
- 角度变化小于30度的两个同景别剪辑会产生"跳切"（jump cut）
- 观众会感觉像"定格的画面突然跳了一下"

### 2.3 AI的跳切问题
AI多段视频拼接时，即使镜头描述不同，生成的画面也可能角度极为接近，产生跳切。

**解决方案**：在分镜设计中明确标记机位角度变化。

## 三、视线匹配（Eye-Line Match）

### 3.1 定义
角色看的方向必须与前一个镜头中该角色注视的方向匹配。

### 3.2 视线的连续性规则
- A看画面右边 → B必须在画面左边回看A（符合180度轴线）
- A看画面上方 → B在俯视A

### 3.3 AI的视线问题
AI生成的单图眼神通常很美，但多段视频中角色的视线方向经常漂移。

**解决方案**：用"视线向量"明确标记——`looking screen left` 或 `looking screen right`。

## 四、跳动接续（Match Cut / Action Match）

### 4.1 定义
利用形状、动作或声音的相似性连接两个镜头。

### 4.2 三种匹配剪辑

| 类型 | 描述 | 经典案例 |
|---|---|---|
| 动作匹配（Action Match） | A的动作延续到B完成 | 扔骨头→飞船旋转（《2001太空漫游》） |
| 图形匹配（Graphic Match） | 形状相似的画面连接 | 旋转的陀螺→旋转的舞者（《盗梦空间》） |
| 音效匹配（Sound Match） | 声音桥接 | 关门声→炮火声 |

### 4.3 文档中的应用
- **J-Cut**：地牢极寒的锁链刮擦声在石林段尾提前进入
- **冰火硬切**：不靠匹配剪辑，靠极端反差建立Scene-First

## 五、剪辑节奏（Editing Rhythm）

### 5.1 节奏分类

| 节奏 | 平均镜头时长 | 情绪效果 |
|---|---|---|
| 极慢 | 10-30s | 沉思、压迫、等待 |
| 慢 | 5-10s | 情绪积累、沉浸 |
| 适中 | 3-5s | 标准叙事 |
| 快 | 1-3s | 动作、紧张 |
| 极快/蒙太奇 | 0.5-1s | 慌乱、信息轰炸 |

### 5.2 文档中的节奏设计

**全集张弛节奏总谱**（来自《炎龙新娘》文档）：
```
[B01-B03 逃亡拉锯] ────── 密度：高 (10镜) ──── 节奏：↗↗↗
[B04-B05 恶龙截获] ────── 密度：爆发 (12镜) ── 节奏：↗↗↗↗
[B06-B07 地牢祭坛吸血] ── 密度：稳/慢 (7镜) ── 节奏：↘↘
[B08-B09 耳光与真相] ──── 密度：中 (8镜) ──── 节奏：↗↗
[B10-B11 代嫁宣判/强撕] ─ 密度：高密 (11镜) ── 节奏：↗↗↗
[B12-B13 恐惧加码/揪发] ─ 密度：压力 (8镜) ── 节奏：↗↗↗↗
[B14 强灌紫药死钩] ────── 密度：特写窒息 (5镜) ─ 节奏：↘停
```

### 5.3 AI节奏控制的难点
AI无法理解"节奏"的概念——它生成的每个镜头独立来看可能是好的，但连在一起没有节奏感。

**解决方案**：
1. **在分镜表中预设每镜时长**（文档的做法：每镜精确到0.5s）
2. **按节奏需求分组**：高密段→短镜+快切 | 压力段→长镜+慢节奏
3. **尾气口设计**：每段台词结尾预留0.5-1.5s的情绪呼吸空间

## 六、屏幕方向（Screen Direction）

### 6.1 规则
角色在画面中的运动方向必须保持连续性。

- 角色从**画左出画** → 下一个镜头应从**画左入画**
- 角色奔向**画右** → 下一个镜头他应该仍在向右运动

### 6.2 文档中的屏幕方向管理
- **X轴**：奈雅从画左向画右奔跑（逃亡矢量）
- **Y轴**：红龙从画上方向下俯冲（权力高危）
- **Z轴**：红龙远景 → 中景焦石 → 前景奔跑（景深调度）

### 6.3 方向反转（Intentional Reversal）
当你想表达：
- 角色迷路了 → 让他改变方向
- 角色被击退 → 让他从画右退回到画左（反转逃亡向量）

## 七、叙事时间与屏幕时间的匹配

### 7.1 三种时间关系

| 关系 | 定义 | 示例 |
|---|---|---|
| 实时（Real Time） | 屏幕时间 = 叙事时间 | 地牢对话的一分钟 |
| 压缩（Ellipsis） | 屏幕时间 < 叙事时间 | 快切蒙太奇 |
| 延长（Overlap） | 屏幕时间 > 叙事时间 | 慢动作、多角度重复 |

### 7.2 文档中的时间管理
- **实时**：地牢对话段的音画双账本校验（台词时间=画面时间）
- **压缩**：SC01逃亡段（15s浓缩被猎杀的全过程）
- **悬置**：集尾强灌紫药（5镜14.0s，把0.5s的吞咽拉长为5个微镜头）

## 八、视觉节奏与情绪升降

### 8.1 情绪—节奏映射

| 情绪 | 镜头长度 | 运镜特征 | 剪辑频率 |
|---|---|---|---|
| 极度恐慌 | 1-2s | 手持/晃动 | 极高 |
| 紧张 | 2-3s | 推镜/快摇 | 高 |
| 压迫 | 3-5s | 固定/慢推 | 中 |
| 绝望 | 5-10s | 静止/无运动 | 低 |
| 死寂 | 10s+ | 完全静止 | 极低/定格 |

### 8.2 文档中的节奏映射
```
恐慌 → 高密快切 → 短镜(1-2s) → 手持/横移 → SB01a(逃亡)
压迫 → 标准密度 → 中镜(3-5s) → 固定/慢推 → SB02b(耳光)
绝望 → 慢/长 → 长镜(5-10s) → 静止 → SB02e(揪发)
死寂 → 定格 → 特写硬锁 → 零运镜 → SB02f(灌药->定格)
```

## 九、AI导演的视觉语法规则总结

```
1. 每场戏：必须建立空间锚（Scene-First原则）
2. 每段戏：必须建立轴线（A侧/C侧/B侧，严禁无过渡越轴）
3. 每段对话：视线方向必须匹配（eye-line match）
4. 每个角色：位置+姿态+情绪必须连续（快照继承）
5. 每句台词：时间容器必须闭合（音画双账本）
6. 每次物理接触：必须规避AI死穴（蒙太奇替代法）
7. 节奏变化时：密度模式必须切换（高密↔标准↔慢）
8. 情绪到达顶点时：节奏骤停而非渐弱（尾气口设计）
9. 镜头组接时：30度规则防止跳切
10. 跨集衔接时：Series_Map连续性档案必须更新
```
