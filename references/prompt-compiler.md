# 提示词编译与冲突消解

## 1. 指令优先级

按以下顺序解决冲突：

1. 用户明确要求与原始素材事实；
2. 原台词、事件因果和角色身份；
3. 安全、时长和输出格式；
4. 连续性账本与参考图绑定；
5. 镜头、表演、光影和声音设计；
6. 默认风格与美术增强。

低优先级不得改写高优先级。无法同时满足时，保留高优先级，并用最小必要说明指出取舍。

用户明确要求之间也可能互相冲突。遇到“原台词逐字保留 + 15秒单条不拆分 + 最低自然时长超过15秒”等不可满足组合时，不擅自选择：计算保守最低时长，给出缩短台词、延长时长或允许拆分三个选项。

## 2. 可执行性过滤

保留会改变画面的信息：谁、在哪里、何时、做什么、如何移动、看向何处、持有什么、光从哪里来、摄影机如何观察、声音从哪里来、最后停在哪里。

删除或替换：

- 无可观察结果的形容词堆叠；
- 同义重复的画质词；
- 相互冲突的景别、焦段、机位、时间、天气、服装和光源；
- 同一时间不可能完成的动作；
- 模型无法稳定维持且不服务剧情的微小装饰。

不得在压缩时删除或改写以下执行锚点：

- `CHAR-xxx`角色ID、独立参考图编号与首次身份声明；
- `USE`镜头及其视频条目、时间范围和原台词；
- 当前镜头需要的 `REFERENCE-ONLY` 身份、空间、动作起点或尾帧锚点；
- CAM分段节点的实际运动顺序、转向后方向和最终停止构图；
- 道具持有者、左右手、空间轴线、主光和前后镜继承状态。

`SKIP` 镜头、弃用原因、导演审阅说明、完整状态账本、分镜制作标注和质量检查过程不得进入最终视频提示词。

## 3. 复杂度预算

单镜优先只包含：一个主要叙事目标、一个主动作链、一次主要运镜或固定机位、一个焦点变化、一个明确结束构图。

若同时出现多人复杂动作、长台词、高速运镜、环境破坏和多次焦点切换，按顺序降载：

1. 删除无功能背景角色与装饰动作；
2. 固定摄影机或简化运镜；
3. 拆分动作链；
4. 在自然语义点拆成连续多条；
5. 保留剧情与台词，不牺牲因果来凑时长。

## 4. 正向与负向约束

正向描述具体目标状态，负向约束只列高风险失败，不重复整份规则。负向词必须与镜头风险相关；例如无道具镜头无需堆叠道具错误列表。

## 5. 最终压缩

输出前逐句询问：删除这句话是否会改变构图、动作、身份、连续性、声音或交付格式？若不会，删除。优先使用具体名词和动词，避免把提示词写成宣传文案。

## 6. 分镜到视频提示词的编译

按每个 `USE` 镜头编译，不把整张分镜板逐字复制给生成端：

1. 读取镜头使用清单，排除全部 `SKIP`；
2. 为当前视频条目选取 `USE` 镜头和它们依赖的 `REFERENCE-ONLY` 锚点；
3. 首次出现角色时写 `CHAR-001（角色名，参考图片1）`，多人同框逐人声明；
4. 把 CAM 路线节点转换为按时间排序的自然语言运镜，例如“0–2秒低机位缓慢前推；2–4秒顺时针环绕90度；4–4.5秒停顿；4.5–6秒升高；6–8秒后拉并停在双人全景”；
5. 保留路线结果与顺序，不把 `CAM-A0`、颜色、箭头线型或分镜图例当成最终画面文字；
6. 写入角色动作、视线、焦点、道具、光影、声音、转场和尾帧；
7. 复核提示词时间线与分镜镜头号一一对应，再做最终压缩。
