# 提示词编译与冲突消解

## 1. 指令优先级

按以下顺序解决冲突：

1. 用户明确要求与原始素材事实；
2. 原台词、事件因果和角色身份；
3. 安全、时长和输出格式；
4. 连续性账本与参考图绑定；
5. 镜头、表演、光影和声音设计；
6. 默认风格与美术增强。

低优先级不得改写高优先级。无法同时满足时，保留高优先级，并用最小必要说明指出取舍。

用户明确要求之间也可能互相冲突。遇到“原台词逐字保留 + 15秒单条不拆分 + 最低自然时长超过15秒”等不可满足组合时，不擅自选择：计算保守最低时长，给出缩短台词、延长时长或允许拆分三个选项。

## 2. 可执行性过滤

保留会改变画面的信息：谁、在哪里、何时、做什么、如何移动、看向何处、持有什么、光从哪里来、摄影机如何观察、声音从哪里来、最后停在哪里。

删除或替换：

- 无可观察结果的形容词堆叠；
- 同义重复的画质词；
- 相互冲突的景别、焦段、机位、时间、天气、服装和光源；
- 同一时间不可能完成的动作；
- 模型无法稳定维持且不服务剧情的微小装饰。

不得在压缩时删除或改写以下执行锚点：

- `CHAR-xxx`角色ID、独立参考图编号与首次身份声明；
- `USE`镜头及其视频条目、时间范围和原台词；
- 当前镜头需要的 `REFERENCE-ONLY` 身份、空间、动作起点或尾帧锚点；
- CAM分段节点的实际运动顺序、转向后方向和最终停止构图；
- 道具持有者、左右手、空间轴线、主光和前后镜继承状态。

`SKIP` 镜头、弃用原因、导演审阅说明、完整状态账本、分镜制作标注和质量检查过程不得进入最终视频提示词。

## 3. 复杂度预算

单镜优先只包含：一个主要叙事目标、一个主动作链、一次主要运镜或固定机位、一个焦点变化、一个明确结束构图。

若同时出现多人复杂动作、长台词、高速运镜、环境破坏和多次焦点切换，按顺序降载：

1. 删除无功能背景角色与装饰动作；
2. 固定摄影机或简化运镜；
3. 拆分动作链；
4. 在自然语义点拆成连续多条；
5. 保留剧情与台词，不牺牲因果来凑时长。

## 4. 正向与负向约束

正向描述具体目标状态，负向约束只列高风险失败，不重复整份规则。负向词必须与镜头风险相关；例如无道具镜头无需堆叠道具错误列表。

## 5. 最终压缩

输出前逐句询问：删除这句话是否会改变构图、动作、身份、连续性、声音或交付格式？若不会，删除。优先使用具体名词和动词，避免把提示词写成宣传文案。

## 6. 分镜到视频提示词的编译

按每个 `USE` 镜头编译，不把整张分镜板逐字复制给生成端：

1. 读取镜头使用清单，排除全部 `SKIP`；
2. 为当前视频条目选取 `USE` 镜头和它们依赖的 `REFERENCE-ONLY` 锚点；
3. 首次出现角色时写 `CHAR-001（角色名，参考图片1）`，多人同框逐人声明；
4. 把 CAM 路线节点转换为按时间排序的自然语言运镜，例如“0–2秒低机位缓慢前推；2–4秒顺时针环绕90度；4–4.5秒停顿；4.5–6秒升高；6–8秒后拉并停在双人全景”；
5. 保留路线结果与顺序，不把 `CAM-A0`、颜色、箭头线型或分镜图例当成最终画面文字；
6. 写入角色动作、视线、焦点、道具、光影、声音、转场和尾帧；
7. 复核提示词时间线与分镜镜头号一一对应，再做最终压缩。

## 7. 角色、分镜与API素材绑定

生成视频提示词前，把执行素材绑定清单视为唯一编号来源：

1. 每个 `USE` 镜头枚举出场角色ID，并确认每个ID只对应一张独立角色板记录；
2. 每张执行分镜页写明唯一 `VID-ID/PAGE-ID` 和覆盖的USE镜头范围，不绑定其他VID；
3. 当前VID按“必要角色板 → 当前执行分镜页 → 必要场景板 → 连续性尾帧”的实际提交顺序编号为图片1、图片2……；
4. 视频提示词的【图片绑定】逐项复写同一编号、ID与用途；API `content` 按完全相同顺序追加素材；
5. 若删除、替换或重新排序素材，重新生成整个编号清单并复核提示词，不做局部口头修补；
6. `SKIP`、弃用页、跨条总览和文档装饰图永不进入执行绑定清单。

### 7.1 宫格单格选择与路线层剥离

标准故事板、九宫格、二十五宫格和智能宫格使用统一映射：

`PANEL-ID → Sxx/Sxx-Kxx → USE状态 → VID时间段 → CLEAN-ASSET-ID → REVIEW-ASSET-ID → 角色/场景/尾帧依赖`

1. 根据小说/剧本的剧情目标和当前VID选择真正需要生成的USE单格，不机械生成整板全部格子；用户点名“第N格、镜头Sxx、某秒或某画面”时先定位唯一PANEL-ID；
2. 从同PANEL-ID的REVIEW层读取CAM/ACT/GAZE/FOCUS/LIGHT/READ路线与节点，把它们编译为按时间排序的摄影机、人物/道具、视线、焦点和光线自然语言；
3. 最终提示词保留路线的动作、方向、速度、节点结果和停止状态，但删除箭头颜色、线型、代码、图例、镜头号、信息栏和排版说明；
4. 视频【素材绑定】只列被选中的 `PANEL-ID-CLEAN`，不得列 `PAGE/PANEL-REVIEW`；如果一个VID需要多个USE镜头，只按时间顺序提交对应CLEAN单格，不带无关格；
5. `Sxx-Kxx` 属于同一连续镜头时，不因选择多个关键状态而新增切镜；其中一格作为首帧/构图锚点，其他状态编译为变化和尾帧；
6. 只有标注图而没有CLEAN源时停止执行并重建干净单格；不得把带箭头、文字、节点或边框的裁剪图冒充视频内容参考。

## 8. 九部门会审到视频提示词

STYLE LOCK、色卡/LUT、多参考素材、复杂摄影/光影/声音/妆造或专业岗位审阅任务，先读取 `references/cinematic-department-review.md` 并通过九项冲突闸门，再按 `templates/compiled-video-prompt.md` 编译：

1. 每个VID只保留一个当前STYLE LOCK；参考来源按色彩、光线、摄影、场景、表演、节奏或声音职能转译，不携带上一VID的镜头、音乐、剧情或负面词；
2. 把世界坐标、摄影机坐标、屏幕坐标、遮挡链和世界光源图编译成无歧义自然语言；
3. 把胶片/传感器感、画幅、镜头投影、焦段/FOV、光圈/景深、焦点、稳定方式和CAM路径写成一个摄影合同；
4. PAL/LOOK/LGT/AUD使用稳定ID管理，但在首次出现时写全自然语言含义；需要色卡时先按 `templates/color-palette-board.md` 生成 `REFERENCE-ONLY COLOR` 资产，提示词同时引用图片编号和关键P编号/HEX/对象；`STRICT-N` 不虚假承诺逐像素HEX一致；宫格分镜只绑定已选PANEL-ID-CLEAN，REVIEW路线层必须先转译再剔除；
5. 最终顺序固定为“任务与输出 → 素材绑定 → STYLE LOCK → 摄影合同 → 场景/人物/空间 → 表演与生命感 → 时间轴 → 环境光/人物光 → 色卡/调色 → 妆造 → 声音/混音 → 尾帧 → 高风险负面约束”；
6. 环境、人物外观、风格、表演、色卡和声音基线各写一次；时间段只写动作、摄影、焦点、光影、声音和状态变化，不逐段复制全部基线；
7. 会审表、冲突日志、部门意见和长篇制作说明不得进入生成端。

## 9. 提示词卫生检查

编译后执行一次跨段落扫描：

- 删除其他VID、其他色卡、其他角色和其他场景的残留词；
- 合并重复环境光影、胶片、画幅、BGM和负面说明；
- 检查同一概念是否同时出现肯定与否定版本，例如固定/手持、高/低饱和、BGM/无BGM、无人声/哼唱、正顶视/地平线；
- 检查正文是否引入色卡外有意色相、未绑定参考图或无主体归属的声音；
- 色卡图片存在时检查其无UI、无其他VID色板，并明确不把色块、文字、HEX、边框或版式渲染进成片；
- 保留原台词、资产绑定、时间、空间、遮挡、摄影、光色、声音与尾帧的可观察结果。

## 10. 长提示词优化流水线

按以下顺序优化用户原始提示词，不直接在原文上零散删句：

1. **拆块**：识别平台/UI尾注、素材绑定、STYLE、摄影、场景、人物、空间、时间轴、光色、妆造、声音、尾帧和负面约束；
2. **清洗**：删除按钮文案、模型切换提示、图片编辑工具列表、加载状态、生成次数/积分、无关分辨率尾注和复制时混入的其他镜头文字；
3. **建事实表**：把角色、道具、场景、参考图、音频、时长、镜头数量和原台词登记为不可静默改写的事实；
4. **归一术语**：区分胶片/传感器、焦段/FOV、镜头投影、画幅、光圈/景深、支撑方式、运镜和输出分辨率；不要把“35mm”同时当胶片和焦段；
5. **风格分工**：每个电影/创作者/作品参考只绑定色彩、光线、摄影、场景、表演、节奏或声音中的明确职能；相同职能只保留一个主参考和必要辅助特征；
6. **冲突解算**：按指令优先级解决高/低饱和、固定/手持、正顶视/地平线、自然光/practical、鱼眼/普通直线投影、BGM/无BGM、无人声/哼唱、写实主体/绝对NOT CGI等互斥项；
7. **时间编译**：把事件按真实先后放入0.0–T_video；同段并发只保留一个第一视觉主体，其他事件写成背景或声音，不把所有动作都写成同等重点；
8. **基线去重**：角色全貌、场景、STYLE、摄影、环境光、色卡、妆造和声音基线各写一次；时间段只写开始、变化、同步和结束状态；
9. **生成端转译**：把内部ID第一次展开为自然语言，把CAM/ACT/FOCUS/LIGHT/AUD节点变成时间排序描述；制作安全、部门评论和图例不进入成片；
10. **回归核对**：逐项对照原台词、资产编号、PAL色号、镜头数、时长、遮挡、尾帧和用户硬约束，确认优化没有改变剧情或身份。

## 11. 语句级压缩

- 每个句子优先使用“主体 + 可观察动作/状态 + 空间/时间限定”；一个代词不能指向多个角色；
- 中英文同义词只保留一种主表达；只有生成端确实受益的专业术语可在中文后保留一次英文，不连续堆叠翻译；
- 删除Markdown粗体、宣传口号、重复感叹、无画面结果的“极致/震撼/大师级/完美”和与实际输出不符的“8K”承诺；
- 画质描述只保留能改变成片的介质、颗粒、反差、高光响应、材质、空气和运动特征；
- “NOT CGI/NOT 3D”不得删除幻想机械、变形或3D模式主体；改写为“实体材质与实拍摄影观感、避免塑料渲染痕迹”等可观察目标；
- 负面约束由正向目标反推，只保留3–8个最高风险失败；不同时否定用户明确要求的镜头、声音、角色或风格；
- 参考图使用“图片N → 对象 → 锁定属性 → 可变化属性 → 适用时间/镜头”，不得只写“严格按图”；
- 色卡使用“图片N/Pxx/HEX → 职能 → 对象 → 允许受光变化”，不得只写“不可偏色”；
- 对白逐字保留，情绪、方言、音色和空间属性放在说话人首次出现处；同一角色连续说话不重复完整音色说明；
- 删除重复的环境光影段、重复STYLE LOCK、重复画幅/胶片和重复无字幕/无水印声明。

## 12. 输出参数与创作描述隔离

把“Seedance 2.0、1080p、16:9、5s”等已确认参数放在【任务与输出】，不要在每个时间段重复。把编辑界面中的模型名、比例按钮、质量档、图片数量、积分或字符计数视为UI元数据；只有用户明确把它们设为交付要求且当前生成端支持时才保留。

“8K、4K、高清、HD”不能替代真实输出分辨率。若用户用“8K”表达细节目标但实际输出为1080p，把它转译为可观察的纹理、边缘、材质和远近层次，不虚构输出像素。

## 13. 优化后差异摘要

导演审阅层可简短列出：

- `KEPT`：原台词、剧情、身份、资产、色卡、关键镜头和尾帧；
- `MERGED`：重复STYLE、环境光影、中英同义词和通用负面项；
- `RESOLVED`：已按明确优先级解决的摄影、光色、声音或时间冲突；
- `REMOVED`：UI残留、其他VID信息、不可执行宣传词和无关参考；
- `ASK`：无法在不改变用户主要意图的情况下自行选择的硬冲突。

差异摘要不提交生成端。最终提示词仍严格使用 `templates/compiled-video-prompt.md`。

## 14. 强电影质感编译

不要直接追加“电影感、大片感、顶级、极致”等词。按镜头功能从下列层中选择会改变结果的少量锚点，并用因果连接：

1. **镜头可读性**：第一视觉主体、景别、前中后景、遮挡、焦点落点和尾帧；
2. **摄影行为**：主投影、焦段/FOV、主体距离、支撑方式、运镜动机、加减速和停止构图；
3. **曝光与光线**：世界光源、人物主/辅/轮廓职责、面部可读性、黑位、高光响应和空气介质；
4. **色彩与材质**：PAL绑定、肤色/发色保护、表面粗糙度、磨损、湿润、反射和胶片/显示响应；
5. **时间与物理**：自然运动模糊、重量、惯性、接触、脚底受力、衣发/烟尘余韵；
6. **声音透视**：声源、同步、距离、遮挡、动态峰值和收束。

只有叙事需要时才使用浅景深、手持、慢动作、光晕、雾、镜头呼吸或大幅运镜。它们是工具，不是电影质感的默认叠加包。镜头越短，越应减少互相争夺权重的风格词。

## 15. 人物生命感编译

为主表演者建立 `PERF-ID`，先在导演层记录：`目标 → 阻碍 → 外在策略 → 潜台词 → 刺激/感知 → 决定 → 反应延迟 → 可见动作 → 新状态`。生成端只保留当前景别和时间可见的部分：

- **面部**：明确视线落点，选择眨眼、眼睑张力、眉间、鼻翼、下颌、嘴角或吞咽中的少数信号；不让所有五官同时夸张变化；
- **身体**：用呼吸、肩颈张力、脊柱方向、支撑脚、重心转移和手指/抓握表达状态；动作要有预备、发起、停止和余韵；
- **互动**：触碰前有距离变化，接触时有受力，接触后有反作用；视线落在真实对象上，角色不预知尚未发生的刺激；
- **对白**：原台词逐字保留；语气与速度之外，写说话前气息、口型/声源归属、关键词动作和说完后的微反应；未说话者继续倾听但不抢戏；
- **背景生命**：只保留与环境和剧情相关的低幅度行为，不用所有人同步运动制造“热闹”；
- **景别预算**：特写用脸与气息，中景用手部/姿态/互动，全景用路线/尺度/群体关系；不可见细节不写。

真人模式强调自然不对称、皮肤微纹理、真实受光和演员式反应；3D模式强调身份稳定的面部形变、眼神聚焦、呼吸、重量、接触和衣发次级运动。禁止用“表情丰富、动作自然”替代上述可观察链条。
