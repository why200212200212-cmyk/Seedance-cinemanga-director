# 角色多视角设计与差异化规则

## 1. 与分镜流程的关系

先完成剧情分析和分镜形式决策，再锁定必要角色的多视角设计，随后生成详细分镜图和最终提示词：

1. 剧情分析；
2. 选择标准故事板、九宫格、二十五宫格或智能宫格；
3. 角色多视角设计与身份锁定；
4. 详细分镜图；
5. 导演方案与可复制提示词。

角色设计只能补足美术身份，不得反向增加、删减或改写剧情。分镜图中的角色必须继承角色板，不能逐格重新设计。

## 2. 多视角角色板

为每个需要出镜的具名或可识别角色分别建立一张独立角色板。独立是硬规则：一名角色对应一个角色ID、一张运行时角色板和一个参考图编号；不得把多人合并图作为 Seedance 2.0 的身份绑定依据。

- 必选视角：正面、左侧面、右侧面、背面；
- 建议视角：左/右四分之三正面、四分之三背面；
- 全部视角保持同一身份、年龄感、脸型、五官比例、发型、体型、服装层、材质、配饰和标志性细节；
- 使用中性站姿、统一焦段感、统一光线和纯净背景，避免透视、表情或光影造成假差异；
- 不在同一角色板混入其他人物的脸、服装或配色。

## 3. 角色身份注册表

剧情分析后建立并持续复用：

```text
【角色身份注册表】
- CHAR-001｜角色名：[姓名]
  - 独立角色板：[图片1]
  - 身份锚点：[脸部外轮廓、眉眼、鼻形、唇形、下颌、年龄感、稳定标志]
  - 固定外观：[发型、体型、服装层、材质、配饰]
  - 适用镜头：[S01、S03……]
  - 禁止混淆：[不得继承CHAR-002的哪些特征]
```

绑定规则：

- 分镜格在人像附近写 `CHAR-001`，格下信息栏重复“角色名 / 图片编号”；
- 图片提示词写“图片1只绑定CHAR-001”，不得使用“左边的人”“穿黑衣的人”等临时称呼代替身份；
- 视频提示词首次出现时写“CHAR-001（角色名，参考图片1）”，后续保持相同ID；
- API `content` 按绑定清单顺序提交，提示词中的图片编号必须与数组顺序一致；
- 多人同框时逐人列出ID、参考图、站位、朝向、动作和视线，禁止合并成“众人”或依靠模型自行匹配；
- 若平台参考图数量不足，按当前条实际出镜角色优先提交，不能把多名角色拼成一张图冒充独立身份资产；
- 关系总览图和角色差异化对比图只能用于审阅或文档装饰，不能替代独立角色板。

## 4. 角色面部区分矩阵

在内部为任意两名角色做两两比较，至少检查：

- 脸部外轮廓与长宽比；
- 眉形、眼裂方向、眼距与眼窝深浅；
- 鼻梁、鼻头与鼻翼结构；
- 唇形、嘴角和人中；
- 颧骨、面颊体积与下颌线；
- 年龄感、肤色/肤质、雀斑、痣、疤痕等稳定特征。

不同角色至少在多个面部维度形成结构性差异。发型、服装、妆容、配饰和颜色只能加强区分，不能作为唯一差异。禁止复制同一张脸后仅更换发型或服装。

若两名角色在正面、四分之三侧面或侧面仍容易互认错误，判定为相似度过高；先重做其中一人的面部结构，再进入后续生成。

## 5. 剧情例外

仅在原剧本明确要求以下关系时允许控制性相似：

- 双胞胎或多胞胎；
- 克隆体、复制人或同源人造身份；
- 明确要求高度相似的血缘人物；
- 替身、伪装或剧情指定的相貌相似者；
- 同一人物的不同年龄、时间线或身份版本。

即使属于例外，也要列出剧情允许的最小可辨识差异，例如年龄痕迹、痣/疤位置、眼神习惯、面部肌肉状态、发际线、体态或服装身份标记。剧本未明确时，不得自行添加亲缘或相似设定。

## 6. 一致性与差异化双重门槛

- 同一角色：跨视角、跨镜头、跨场景和跨视频保持同一身份；
- 不同角色：在同一剧本中保持明确、稳定、可复核的外貌差异；
- 参考图：一图一角色绑定，禁止混脸；若用户提供的不同角色参考图本身高度相似，先提示风险并在不违背参考图的前提下强化非核心差异；
- 质检失败：停止进入最终提示词，先修正角色板。
