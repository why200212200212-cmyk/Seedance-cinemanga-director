# 九部门导演会审与镜头统一合同

## 1. 模块边界

在剧情、必要人物、角色身份、场景结构、VID时长和USE镜头已经确定后使用本模块。它不新增剧情、角色、道具、风格或镜头，只让导演、视觉构造、摄影、灯光、调色、化妆、人物妆造、声音设计和混音共享同一份可执行镜头事实。

以下任一情况出现时，必须完整读取本文件并使用 `templates/cinematic-department-review.md`：

- 用户提供 `STYLE LOCK`、电影/导演/摄影参考、LUT、胶片或色卡；
- 同时绑定多张角色、服装、道具、场景、分镜、音色或尾帧参考；
- 使用一镜到底、复杂运镜、特殊镜头、遮挡转场、变形或多人调度；
- 镜头包含对白、环境音、动作音、BGM或多个空间声源；
- 用户明确要求从摄影、光影、调色、妆造、声音或导演等专业岗位审阅。

普通低复杂度镜头也执行本文件的冲突闸门，但只保留必要字段。会审表属于导演审阅资料，不得整表粘贴到生成端。

## 2. 唯一事实与作用域

先建立唯一镜头事实表，再让各部门在其上工作。禁止每个部门各写一套人物、空间、时间或风格事实。

- `VID-ID / Sxx / t_start–t_end`：唯一时间范围；
- `CHAR-ID / LOOK-ID`：人物身份与当前妆造版本；
- `PROP-ID / SET-ID`：道具与场景；
- `CAM-ID`：摄影机、镜头、投影和路线；
- `LGT-ID`：世界中的自然光、实用光、自发光与环境光；
- `PAL-ID / GRADE-ID`：色卡与当前调色版本；
- `AUD-ID / VOICE-ID`：环境、动作、对白、音乐与音色；
- `PERF-ID`：人物目标、阻碍、策略、潜台词、动作与微反应曲线；
- `STYLE-ID`：项目、序列、VID或镜头级风格来源与作用域。

采用四级继承：`PROJECT → SEQUENCE → VID → SHOT`。下级只覆盖明确字段；换场或新VID先声明继续继承哪些字段。每个VID重新编译一次 `STYLE LOCK`，不得复制上一个VID的导演名、作品名、镜头角度、表演节奏、音乐或负面词。

## 3. 三套坐标必须分离

所有站位、光线和运动分别写清：

1. **世界坐标**：门、窗、太阳、角色、道具和路线在场景中的真实位置；
2. **摄影机坐标**：主体相对镜头的前后、距离、朝向和遮挡；
3. **屏幕坐标**：主体在画面左/中/右、上/中/下、占比和出入画方向。

先写世界关系，再推导摄影机关系，最后得到屏幕构图。摄影机移动时，太阳、窗户和固定灯具保持世界坐标不变；不得让光源、人物左右或出入口随画面自动翻转。

建立遮挡链，例如 `前景道具 > CHAR-002 > 背景门`。若某对象必须完全遮住另一对象，写出遮挡比例、允许露出的部位和持续时间；不要只写“挡住”。

## 4. 九部门会审顺序

### 4.1 导演

- 为每个USE镜头只保留一个主要戏剧目标和一个观众注意力中心；
- 写明开场状态、触发事件、情绪/动作反转和结束状态；
- 为承担表演的角色建立 `PERF-ID`：`当下目标 → 阻碍 → 外在策略 → 潜台词 → 感知 → 决定 → 可见反应 → 新状态`；
- 把“紧张、俏皮、震惊、疲惫”等情绪词转成可见行为：视线落点、反应延迟、呼吸、眨眼、吞咽、下颌/肩颈张力、手指动作、重心和说话前后停顿；每个短节拍只选2–4个最能说明状态的信号，避免全脸同时表演；
- 人物在主动作之间保持低幅度生命感，如呼吸、重心微调、倾听和环境反应，但不得随机摇头、漂移视线、无原因挥手或抢夺主叙事动作；
- 对白镜头同时校验说话人的口型、气息和动作，以及未说话者的倾听反应；反应应由前一刺激触发并保留自然延迟，不能预知事件；
- 把电影、导演或作品参考拆成具体职能：色彩、光线、摄影、场景、表演、剪辑节奏或声音；禁止一个参考名自动控制全部部门；
- 参考只提供形式语言，不迁移原作人物、剧情、标志性镜头或无关动作。

### 4.2 视觉构造

- 锁定前景/中景/背景、正负空间、尺度参照、材质层级和视觉动线；
- 为幻想机械、变形和巨物给出接触面、支撑、重量、磨损、运动来源和结束形态；
- 同时出现“人物高度”“柱体倍数”“画幅占比”等尺度时选一个主基准，其他值只能作校验；
- 合成分镜图先生成无字画格，再排版标题、镜头数据、箭头和长说明；不得要求图像模型一次准确绘制大量文字。

### 4.3 摄影

把以下字段分开，不用“35mm电影感”代替全部规格：

`拍摄介质/传感器感 → 画幅 → 镜头类型/投影 → 焦段或视场角 → 光圈/景深 → 主体距离 → 焦点计划 → 快门/运动模糊 → 支撑方式 → CAM路径 → 停止构图`

- 胶片规格不等于镜头焦段；
- `truck` 是横向平移，`dolly` 是靠近/远离，`pan/tilt` 是原地转动；
- 固定机位可记录载具或地面传导的受控微震，但不得同时写自由手持；
- 严格正顶视图通常不出现地平线；鱼眼球形投影与传统变形宽银幕效果同时出现时指定主光学语言；
- 景深必须与焦段、光圈、距离和焦点计划相容；需要表情或口型可读时不得无意把人物置于不可读虚焦区。
- 摄影机运动必须有叙事动机：揭示、跟随、压迫、释放、连接视线或重组空间；不能只为“电影感”持续漂移。运镜起停、加减速、地面/载具传导和演员节拍必须同步。

### 4.4 光影

建立世界光源图：

`LGT-ID → 类型（自然/实用/自发光/环境反射）→ 世界位置 → 方向 → 色温/色相 → 软硬 → 相对强度 → 影响对象 → 阴影方向 → 是否连续`

- 区分环境光与人物光；环境成立不代表面部、眼睛、头发和服装可读；
- 人物至少检查主光、填充、轮廓、眼神高光或面具屏幕反射；
- `practical light only` 不包含太阳和天空；若使用夕阳，写“自然光+有依据的实用光”；
- 冷色轮廓、体积光和发光边缘必须有来源；雾、尘、雨和烟只显示已有光路，不自行发光；
- 遮挡转场同步检查曝光、反射和声音透视，避免画面变黑但环境仍像完全敞开。

### 4.5 视频调色

为每个色卡建立 `PAL-ID`，把每个HEX绑定到以下一种职能：

`固有色 / 光源色 / 阴影倾向 / 高光倾向 / 自发光 / 强调色 / 中性基底`

色卡模式分两类：

- `GUIDED`：允许在同一色相家族内随光照、曝光、雾和胶片响应变化；
- `STRICT-N`：不得新增有意设计色相，但仍允许受光后的明度、饱和度和邻近色偏变化。

执行顺序为：`固有色与光源角色 → 曝光/白平衡 → 光照与空气影响 → 胶片/显示风格 → 输出色域`。HEX是中性参考或设计锚点，不承诺成片逐像素相等。

- 给出主色/辅色/强调色的大致占比；
- 检查正文是否引入色卡外HEX；
- 检查高饱和与低饱和、暖调与冷调、低反差与深黑是否互相冲突；
- 人物肤色与发色必须从环境综合色中分离；
- 肤色保护不是把人物照白：保留皮肤明暗、血色、微纹理和材质高光，只避免综合色吞没身份与表情；
- 色卡截图必须裁掉按钮、加载浮层、边框和无关段落色板；
- `GRADE-ID` 只继承到明确指定的场景，不把上一段LUT或色调带入新场景。

需要生成可视色卡时，使用 `templates/color-palette-board.md`：先把结构化PAL表作为唯一真源，再生成无UI的标注PNG。色卡在当前VID执行素材清单中标为 `REFERENCE-ONLY COLOR`；视频提示词同时引用色卡图片编号和关键P编号/HEX/绑定对象，并明确不得把色块、文字、边框或版式渲染进成片。

### 4.6 化妆

- 锁定肤质、妆面光泽、面部标记、汗水、灰尘、伤妆和随时间变化；
- 在彩色光、极浅景深和高反差下检查五官、口型、贴纸、眼妆与年龄感是否仍可读；
- 面部标记记录左右侧、精确位置、形状、颜色和版本，禁止换脸、镜像或变形；
- 真人涉及换装、身体遮挡或特殊妆效时，只描述银幕可见结果、遮挡和连续性；现实拍摄交由合格服化、现场安全及相关专业人员执行。

### 4.7 人物妆造

- 为每人建立 `LOOK-ID`：发型/发色、服装分层、材质、鞋袜、配饰、道具、机械义体、污损和禁止变化；
- 把身份锚点分为轮廓、颜色、材质和局部标志四类，避免只靠服装颜色认人；
- 跑动、转身、变形、脱穿和强风镜头逐项检查配件、左右肩、衣层和发型是否能保持；
- 参考图绑定使用“锁定属性/可变化属性/适用镜头”，不得把服装、道具、妆面或材质迁移到其他角色。

### 4.8 听觉构造

建立声音事件图：

`AUD-ID → 时间 → 声源主体 → 叙事内/叙事外 → 环境/动作/拟音/对白/非语言人声/BGM → 空间位置 → 动作同步点 → 开始/结束状态`

- 把“无人声”拆成无对白、无旁白、无非语言人声三项；哼唱属于非语言人声；
- 把白噪音、持续低音和配乐明确归类，禁止同时声明“唯一BGM”和另一套BGM；
- 物体遮镜、远离、进入室内或贴住镜头时，同步改变声音距离、混响、高频和声压；
- 每个声音必须有主体归属；数量不同的机械足、角色和道具不得共享模糊声源。

### 4.9 调音

只规定叙事混音关系，不虚构生成端不能保证的响度数值：

- 排列优先级：关键对白/口型 > 当前叙事动作 > 必要环境 > 音乐与装饰层；无对白镜头可按剧情重新排序；
- 为对白让出中频，为撞击保留短时动态，为持续引擎、低吟、铜钦和低频冲击分配不同时间或频段重心；
- 标记需要ducking、遮挡变闷、远近变化、左右/前后空间定位和混响切换的位置；
- 禁止所有声音持续满响；动作完成后让声音进入余韵或静默，为尾帧留出听觉收束。

## 5. 时间与复杂度闸门

把每个VID拆成可观察事件：台词、表情反应、主要动作、变形、运镜节点、焦点变化、转场和尾帧。给每项分配真实时间，不把并发书写误当成模型可以同时清楚表现。

以下任一情况视为过载：

- 长台词同时发生复杂运镜、身份变形和快速离场；
- 5秒内要求多个完整动作链和两次以上情绪反转；
- 一个镜头包含两种主投影、两条主运镜或多个争夺注意力的主体；
- 声音事件数量超过画面可辨识动作，或对白没有可懂度空间。

降载顺序：保留剧情和原台词 → 删除无功能装饰动作 → 固定或简化摄影机 → 减少焦点/风格变化 → 拆分动作链 → 拆镜或拆VID。不得用“快节奏”“丝滑”“8K”掩盖时间不足。

## 5.1 电影质感与人物生命感闸门

“电影感”不得作为空泛形容词单独提交。至少由当前镜头真正需要的以下层级构成，并按重要性限量选择：

1. **叙事可读性**：观众先看到谁、为什么看、何时发生转折、尾帧留下什么；
2. **表演生命感**：目标、阻碍、潜台词、感知—决定—反应链、呼吸/重心和未说话者反应；
3. **场面与摄影**：前中后景、尺度、遮挡、镜头投影、焦点和有动机的CAM路径；
4. **光色与材质**：有来源的环境光/人物光、曝光层次、肤色保护、PAL/GRADE和可触摸的表面响应；
5. **时间运动**：自然快门观感、惯性、接触、衣发/尘雾次级运动、动作后的余韵；
6. **听觉空间**：声源归属、同步、距离、遮挡、动态和静默。

提示词强度来自“少量高权重锚点 + 明确因果 + 可读时间”，不是把所有电影术语同时堆入。特写优先面部、口型与微反应；中景优先手部、姿态和互动；全景优先路线、尺度和群体关系。超出当前景别不可见的表演细节不占提示词预算。

## 6. 冲突闸门

输出前逐项生成 `PASS / RESOLVE / ASK`：

1. **作用域**：STYLE、PAL、LOOK、LGT、AUD是否来自当前VID，是否混入上一段；
2. **时间**：对白、动作、运镜、变形和尾帧是否放得下；
3. **光学**：机位、地平线、投影、焦段、景深、焦点和稳定方式是否相容；
4. **空间**：世界、摄影机、屏幕坐标及遮挡链是否一致；
5. **光色**：自然光/实用光来源、主光方向、色卡模式和胶片处理是否相容；
6. **人物**：身份、妆面、发型、服装层、配件、左右关系、材质和PERF表演因果是否稳定；
7. **声音**：对白/无人声、BGM/无BGM、声源数量、透视和混音优先级是否一致；
8. **素材**：图片/音频编号、资产ID、适用镜头和API提交顺序是否一致；
9. **提示词卫生**：环境、风格和负面约束是否重复，是否存在不会改变画面的宣传词。

能按用户已给优先级唯一解决的标 `RESOLVE` 并写明舍弃项；会改变剧情、台词、模式、时长或主要风格且无法判断的标 `ASK`，只询问最小问题。全部硬项通过后才进入最终提示词编译。

## 7. 双模式专项

- **3D影视级**：在共享合同后追加绑定、碰撞、变形、材质响应、次级运动、虚拟摄影机和渲染稳定性；不得声称“NOT CGI”同时又要求3D模式。
- **真人影视级**：在共享合同后追加真实设备可行性、演员表演、实体材质、曝光、服化连续性和现场安全边界；幻想物体写成“实拍摄影观感、实体/微缩/机械特效质感、隐藏数字合成痕迹”，不要用绝对 `NOT CGI` 删除剧情主体。

## 8. 输出合同

使用 `templates/cinematic-department-review.md` 生成导演审阅层；需要色卡时再使用 `templates/color-palette-board.md` 生成并校验PAL资产，最后按 `templates/compiled-video-prompt.md` 只把通过会审的结果压缩进当前单条或多条模板：

1. 每个VID只保留一个当前 `STYLE-ID` 和一个摄影合同；
2. 色卡、光源、妆造、空间和声音使用稳定ID，但最终自然语言必须可独立理解；
3. 风格参考按职能翻译为可观察特征，不重复作品/创作者名单；
4. 最终提示词按“任务与输出 → 素材绑定 → STYLE LOCK → 摄影合同 → 场景/人物/空间 → 表演与生命感 → 时间轴 → 环境光/人物光 → 色卡/调色 → 妆造 → 声音/混音 → 尾帧 → 高风险负面约束”排序；
5. `PERF-ID` 编译为一段可独立理解的表演合同；时间轴只写触发、变化和结束状态，不反复堆叠情绪形容词；
6. 色卡作为REFERENCE-ONLY COLOR引用；关键色号及其对象绑定仍写入提示词，色卡版式和标签不得进入成片；
7. 导演会审表、冲突日志、部门意见、分镜长文字和完整负面库不提交生成端。
