# Seedance API 执行契约

仅当用户明确要求实际生成、已配置凭证并理解可能产生费用时，才调用 API。只请求提示词或分镜时不要创建任务。

## 当前接口

- 鉴权：`Authorization: Bearer $ARK_API_KEY`。
- 创建：`POST /api/v3/contents/generations/tasks`。
- 查询：`GET /api/v3/contents/generations/tasks/{id}`。
- 取消：`DELETE /api/v3/contents/generations/tasks/{id}`。
- 创建必填字段：`model`、`content`；模型 ID 或推理接入点 ID由用户账户决定，不在 Skill 内锁死。
- 任务状态：`queued`、`running`、`succeeded`、`failed`、`cancelled`；成功结果读取 `content.video_url`。

## 素材映射

便捷模式把可复制视频提示词作为第一个 `text` 内容项，再按用户给定顺序追加 `image_url`；每个参考图内容项顶层固定写入 `role: reference_image`。普通角色板、分镜页和场景板使用 API 可访问的 HTTPS URL（仅本机环回测试允许 HTTP）；已获授权并进入火山方舟可信素材库的人物可使用官方 `asset://asset-id` URI。提示词中的“图片1、图片2……”必须与该顺序严格一致，不要把本地文件名冒充为模型可见编号，也不要把未授权真人素材伪装成可信资产。

提交前根据当前视频条目的镜头使用清单建立“执行素材清单”：

1. 只选择当前条 `USE` 镜头实际需要的独立角色板、当前分镜页、场景板和上一条尾帧；
2. `REFERENCE-ONLY` 只在服务当前 `USE` 镜头且平台能力允许时提交，不单独占用视频时间；
3. `SKIP` 镜头、弃用页、角色对比装饰图和文档插图禁止提交；
4. 每个角色保持 `CHAR-xxx → 角色名 → 图片编号 → URL/资产URI` 一一绑定，不用多人合并图替代；
5. 提示词首段声明图片绑定，后续时间线按镜头号引用同一角色ID；
6. 参考图超出账户能力时，优先保留当前条出镜角色的独立角色板、当前执行分镜页和连续接力所需尾帧；不拼图冒充独立身份参考。

dry-run 时逐项核对文本中的图片编号、`content` 数组索引和执行素材清单。任何不一致都停止创建任务。

需要首尾帧、视频、音频或其他账户特定的多模态字段时，使用 `--content-json` 透传当前官方 `content` 数组，不猜测字段。先用 `--dry-run` 检查最终请求体。

## 运行边界

- 创建请求不自动重试，避免网络不确定时重复创建付费任务。
- 查询遇到 429 或 5xx 可有限重试。
- API 调用拒绝重定向，避免鉴权头被转发；视频下载只跟随符合 HTTPS/本机环回地址规则的重定向。
- API 密钥只从环境变量或本地 `.env` 读取，不写进提示词、日志、仓库或示例。
- 实际运行前确认模型权限、分辨率、时长、画幅、音频、参考素材数量和账户配额；未知能力不凭空承诺。
- 只把“可复制视频提示词”提交给生成端，不提交导演审阅说明、状态账本或内部检查记录。

完整命令与官方文档入口见 `docs/api-integration.md`。
